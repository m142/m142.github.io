aria2

2016-01-09

用树莓派做下载机，除了速度慢点，其它都挺不错的，功耗低，也稳定。 aria2 是一款非常优秀的小下载工具，支持 http/https, ftp, sftp, bt, metalink 等协议，除了命令行外，也有 web ui，是 Linux 系统上下载的不二选择。

1. 编译安装

下载最新的版本 1.19.3 ，并解压，进入目录并执行以下命令编译安装。

sudo apt-get install autoconf autopoint libtool
./configure
make
sudo checkinstall     # uninstall: `sudo dpkg -r aria2`

2. 配置

aria2 的配置文件是 ~/.aria2/aria2.conf ，所有的配置项说明都可以在 这里 找到。我的配置如下所示。 其中，dir, rpc-listen-port, rpc-secret 等参数可以根据自己需要配置，各参数的含义见 文档 。

log=/tmp/aria2c.log
dir=~/download

# http
check-certificate=false
http-accept-gzip=true
user-agent=uTorrent/2210(25130)

# rpc
rpc-allow-origin-all=true
rpc-listen-port=6800
rpc-secret=secretstring

# download
max-concurrent-downloads=5
continue=true
max-connection-per-server=5
min-split-size=10M
split=10

# bt
bt-enable-lpd=true
bt-max-open-files=256
bt-max-peers=64
bt-prioritize-piece=head=4M,tail=4M
bt-tracker-connect-timeout=16
bt-tracker-timeout=16
peer-id-prefix=-UT2210-

# advanced
#daemon=true    # use `aria2c -D` to start daemon
disk-cache=32M
file-allocation=falloc
log-level=notice

配置文件中之所以没有将 daemon 配置成 true，主要是考虑到有时候需要用命令行下载，如果将 aria2 默认启动为 daemon 则不太方便。同时，如果需要启动为 daemon，则加上 -D 参数即可，非常方便。

3. 常用命令

常用下载命令如下：

  1. http 下载：aria2c http://example.org/mylinux.iso
  2. 多连接下载：aria2c -x2 http://a/f.iso
  3. bt 下载：aria2c http://example.org/mylinux.torrent
  4. Magnet 下载：ria2c 'magnet:?xt=urn:btih:248D0A1CD08284299DE78D5C1ED359BB46717D8C'
  5. 下载文件中的所有链接：aria2c -i uris.txt

更多常用命令见 这里。

4. Web UI

如果只能命令行控制的话，显然不太方便，所以需要有相应的 Web UI。

官方推荐的 UI 是 webui-aria2，功能丰富，界面漂亮，但就是稍显臃肿，有种杀鸡用牛刀的感觉。我用的是 yaaw，界面比较简单，常用功能都有，对我来说比较合适。

如果只是在家里的局域网使用的话，那直接访问 index.html 就行，非常简单。

如果需要在外网访问，则需要一些额外的配置，稍微麻烦一些，下面简要描述。

  1. 配置 DDNS 访问下载机。ISP 分配的 IP 不会一直保持不变，所以需要配置域名访问家里的网络。可以参考之前的 这篇文章。

  2. 配置 Nginx 。如果对安全比较重视的话，可以配置 SSL 证书。相关配置如下，需要修改的地方有 proxy_pass 配置项中端口号，需要和 ~/.aria2/aria2.conf 中 rpc-listen-port 的配置一样。另外，为了防止其他人访问，需要配置访问控制，就是 auth_basic 和 auth_basic_user_file 这两个配置项。关于访问控制的具体介绍和使用方法见 Apache 文档 。最后，SSL 证书可以使用 StartSSl 或 Let’s Encrypt，都可以申请到免费的证书。

    server {
    listen       443 ssl;
    server_name  aria2.example.com;

    ssl_certificate      /etc/ssl/private/example.crt;
    ssl_certificate_key  /etc/ssl/private/examplekey;

    auth_basic "Origin";
    auth_basic_user_file htpasswd;

    location = /jsonrpc {
        proxy_pass http://localhost:6800;
        include proxy_params;
    }

    location / {
        root   /path/to/yaaw;
        index  index.html;
    }
}

最后，访问配置的域名，比如 https://aria2.example.com ，输入配置的访问控制用户名和密码获得访问权限。成功后， 配置 JSON-RPC Path 为 https://token:secretstring@aria2.example.com:443/jsonrpc

其中 secretstring 是 ~/.aria2/aria2.conf 中的 rpc-token 项所配置的 token 。
