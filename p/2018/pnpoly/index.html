PNPOLY: 射线法判断点是否在多边形区域内部

2018-10-22 rd

最近跟地图打交道，有个问题是判断点是否在某个多边形区域内（一般都是凸多边形）。 其实之前也知道一个算法：就是以要判断的点为起点，以区域的顶点为终点作线段或直线， 遍历所有区域顶点，判断直线扫过的角度，如果是360度，说明在区域内，否则就是在区域外。

这次学到了一个新的算法 PNPOLY，代码非常简练，如下：

int pnpoly(int nvert, float *vertx, float *verty, float testx, float testy)
{
  int i, j, c = 0;
  for (i = 0, j = nvert-1; i < nvert; j = i++) {
    if ( ((verty[i]>testy) != (verty[j]>testy)) &&
     (testx < (vertx[j]-vertx[i]) * (testy-verty[i]) / (verty[j]-verty[i]) + vertx[i]) )
       c = !c;
  }
  return c;
}

该算法的核心思想就是：以测试点为起点，作一条水平的射线，判断该射线与多边形区域的边的交点个数，如果为奇数个，说明在区域内，否则就在区域外。 为什么能成立呢？可以这样考虑：多边形区域（不考虑有洞的情况）将空间分成了内外两个部分，如果测试点在空间内，那么作一条射线，至少会与边有一个交点，相交后就到了区域外。 如果再有一个交点，又到了区域内，以此类推。因此奇数个交点说明在区域内， 偶数个说明在区域外。

理解了思想，就是具体实现了。这段代码的详细解释如下：

  1. for (i = 0, j = nvert-1; i < nvert; j = i++) 该 for 循环中 i 和 j 是区域的相邻顶点下标，这样它们就构成了一条边。 首先将 i 设置为第1个顶点， j 为最后一个顶点，每次循环后，i 往后移，j 设置为 i，这样就遍历了所有的边。
  2. ((verty[i]>testy) != (verty[j]>testy)) 以 (testx, testy) 点作一条平行于x轴向右的射线，要与 (vertx[i], verty[i]) 和 (vertx[j], verty[j]) 构成的线段有交点， 必须要满足 testy 在 verty[i] 和 verty[j] 之间
  3. testx < (vertx[j]-vertx[i]) * (testy-verty[i]) / (verty[j]-verty[i]) + vertx[i]) 通过 (vertx[i], verty[i]) 和 (vertx[j], verty[j]) 两点的直线方程为 y - verty[i] = (x - vertx[i]) * (verty[j] - verty[i]) / (vertx[j] - vertx[i])， 稍作变换得到：x = (vertx[j] - vertx[i]) * (y - verty[i]) / (verty[j] - verty[i]) + vertx[i]。 射线要与该直线相交，则 (testx, testy) 点要在该直线的左边，因此需要满足 testx < (vertx[j] - vertx[i]) * (testy - verty[i]) / (verty[j] - verty[i]) + vertx[i]
